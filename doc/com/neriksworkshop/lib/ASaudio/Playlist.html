InheritancePlaylist Group flash.events.EventDispatcher
ImplementsIAudioItem

Playlists are linear Groups. They behave the same way, except they only accept Track objects as children, they only play a Track at once (except for crossfades), and they automatically play children successively.

loopproperty 
loop:Boolean  [read-write]

Determines whether the playlist should repeat or not (default is true). Playlist.loop behaves differently from Group.loop, because when set to true, the whole playlist will repeat (each track is played, and when the last track has been played, the first track starts). If you want to set the loop property for each track inside the playlist, use the Track.loop property.

Constructor detail
Playlist()constructor
public function Playlist(_items:* = null, _name:String = null)

Creates a new Playlist object.

Parameters
_items:* (default = null) — an object containing children you want to add to the group :
  • If you provide a single Track, Group or Playlist object, this object is added as the single child of the group. Use addChild to add other children the same way later.
  • If you provide an Array containing Track, Group or Playlist objects, these objects are added to the group. Use addChild to add each other child individually later.
  • If you provide a string containing the url of a playlist file, the playlist is loaded and children are generated from this playlist. The playlist must have an xspf or xml extension for the XSPF format, or a m3u extension for the M3U format. Use loadXSPF and loadM3U to add tracks the same way later.
  • If you provide an XML object, the latter is interpreted as raw XSPF data, and children are generated from the playlist's items. Use addTracksFromXSPF to add tracks the same way later.
  • If you provide an Array containing strings, each string is interpreted as a track url, and children are generated from these urls. Use addTracksFromUrls to add tracks the same way later.
 
_name:String (default = null) — provide a name if you want to use the Group's method getItemByName(). If you don't provide a name, the name will be created from the auto generated uid.

start()method 
public override function start(_fade:Boolean = false, _startTime:Number = 0, _useStartTimeFromCookie:Boolean = false):void

Starts playback of the playlist, beginning from the first track.

Parameters
_fade:Boolean (default = false) — _fadeIn Fades volume in, using time set by AudioAPI.DURATION_PLAYBACK_FADE.
 
_startTime:Number (default = 0) — The initial position at which playback should start. If _startTime > 1, value is in milliseconds. If _startTime <= 1, value is from 0 (begining of the track) to 1 (end of the track).
 
_useStartTimeFromCookie:Boolean (default = false) — If set to true, the playlist will start at a position saved in a cookie, and previously retrieved on this playlist using cookieRetrieve() (_startTime parameter is then ignored).
startAt()method 
public function startAt(_index:int = 0, _fade:Boolean = false, _startTime:Number = 0, _useStartTimeFromCookie:Boolean = false, _useTrackIndexFromCookie:Boolean = false):void

Starts playback of the playlist, beginning from the track specified in the _index parameter.

Parameters
_index:int (default = 0) — The track index at which the playlist playback should start.
 
_fade:Boolean (default = false) — If set to true, fades in next track, and fades out the currently playing track (if applicable). The fade length is set by AudioParams.DURATION_TRANSITIONS.
 
_startTime:Number (default = 0) — The initial position at which playback should start for the specified track, in milliseconds if _startTime > 1, or if _startTime <= 1, from 0 (begining of the track) to 1 (end of the track).
 
_useStartTimeFromCookie:Boolean (default = false) — If set to true, the playlist will start at a position saved in a cookie, previously retrieved on this playlist using cookieRetrieve() (_startTime parameters is then ignored).
 
_useTrackIndexFromCookie:Boolean (default = false) — If set to true, the playlist will start from the track specified by a cookie. This cookie must have been previously retrieved on this playlist using cookieRetrieve() (_index parameter is then ignored).
